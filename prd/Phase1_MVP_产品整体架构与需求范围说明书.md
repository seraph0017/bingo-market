# Phase1 MVP 产品整体架构与需求范围说明书

## 一、产品概述

### 1.1 核心定位
东南亚合规预测市场平台（知识变现+话题社区），聚焦科技、商业、文化、学术等知识类赛道，严格规避体育赛事比分、政治选举、宗教相关高风险方向。

### 1.2 项目阶段
Phase1 MVP（10-14周，越南首发）

### 1.3 合规红线
- 平台币（知识币）仅支持法币单向充值，永久不设计提现、用户间转账/赠送功能
- 严格执行越南市场日限50万VND、月限500万VND充值限额
- 18+实名注册、多语言用户协议、全链路内容风控机制
- 禁用"赌博""押注""赔率"等敏感词汇，统一使用"预测""知识币""份额""市场共识概率"

### 1.4 技术约束
- 技术底座：Python FastAPI + Vue3 H5 + PostgreSQL 16 + Redis 7
- 架构原则：单体起步、CQRS读写分离、弹性部署、熔断降级
- 交易引擎：Phase1 MVP纯LMSR AMM自动做市
- AI能力：仅调用LLM API，无自研算法

## 二、核心模块拆分

### 2.1 P0核心模块（MVP必做）
1. **钱包与充值支付模块** - 用户资金入口，合规充值控制
2. **LMSR交易引擎核心模块** - 预测市场核心交易逻辑
3. **话题与市场管理模块** - 话题创建、审核、展示
4. **到期结算核心模块** - 市场到期自动结算
5. **纯知识币商城模块** - 知识币兑换商品/服务
6. **用户与权限体系模块** - 18+实名注册、角色权限
7. **运营后台核心模块** - 基础运营管理
8. **内容风控基础模块** - 内容审核、风险控制

### 2.2 模块关联关系
```
用户注册 → 钱包充值 → 话题浏览 → 预测交易 → 到期结算 → 商城兑换
                              ↑          ↑
                          内容风控    运营后台
```

## 三、技术适配要求

### 3.1 数据库设计
- PostgreSQL 16 主数据库，支持JSONB字段存储预测结果
- Redis 7 缓存热点数据：热门话题、市场状态、用户余额
- 读写分离：CQRS模式，写操作通过命令，读操作通过查询

### 3.2 接口设计
- RESTful API + WebSocket 实时推送
- JWT Token 认证，接口权限分级
- 熔断降级：Hystrix模式，异常情况返回兜底数据

### 3.3 部署架构
- 单体应用起步，Docker容器化部署
- Nginx负载均衡，支持水平扩展
- 监控告警：Prometheus + Grafana

## 四、项目排期建议

### 4.1 开发周期
- 总周期：12周（含2周缓冲）
- 核心开发：10周
- 测试验收：2周

### 4.2 模块开发顺序
1. Week 1-2: 用户与权限体系 + 钱包支付模块
2. Week 3-5: LMSR交易引擎 + 话题市场模块  
3. Week 6-7: 到期结算 + 内容风控模块
4. Week 8-9: 纯知识币商城 + 运营后台
5. Week 10-11: 集成测试 + 性能优化
6. Week 12: UAT验收 + 上线准备

### 4.3 关键里程碑
- M1 (Week 2): 用户注册登录、钱包充值完成
- M2 (Week 5): 核心交易流程闭环
- M3 (Week 9): 全功能集成完成
- M4 (Week 12): MVP正式上线

## 五、非功能需求

### 5.1 性能指标
- 接口响应时间：< 500ms (P95)
- 并发支持：1000+ TPS
- 页面加载：首屏 < 2s

### 5.2 可用性
- 服务可用性：99.9%
- 故障恢复：5分钟内自动恢复
- 数据一致性：最终一致性，强一致性关键场景

### 5.3 安全要求
- 数据加密：敏感数据AES-256加密
- 防刷机制：IP限流、行为分析
- 审计日志：全操作留痕，保留180天

## 六、验收标准

### 6.1 功能验收
- 完整的用户注册→充值→预测→结算→兑换闭环
- 所有合规要求100%落地
- 核心业务流程异常处理全覆盖

### 6.2 性能验收
- 压力测试通过1000并发用户
- 数据库查询性能达标
- 缓存命中率 > 80%

### 6.3 安全合规验收
- 通过第三方安全扫描
- 合规顾问审核通过
- 用户协议多语言版本完成
